### Learned Behaviors

- ERA5: use `ecmwf-datastores-client` (NOT deprecated `cdsapi`) ŌĆö the old client had unrecoverable DNS/connection failures
- Advisory lock: use dedicated raw DB connection, NOT xact lock or ORM session ŌĆö ORM commits don't affect it
- Scheduler: use in-memory job store, NOT `SQLAlchemyJobStore` ŌĆö persistent store causes phantom runs from stale `next_run_times`
- Never fill missing data with zeros (GOTCHA Law #1) ŌĆö if data sources fail, skip forecasts
- Never use generic model filenames ŌĆö always use `active_model.json` for dynamic resolution
- Always restart containers after code changes to reload Python modules
- `misfire_grace_time=1` drops missed triggers ŌĆö no catch-up runs
- Startup lock cleanup (`_clear_stale_locks()`) terminates old PostgreSQL backends holding advisory locks
