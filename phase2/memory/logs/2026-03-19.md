## Learned Behaviors Added This Session

**Pipeline Completion Timestamp Trap:** `ForecastLog.issued_at` is the write time at pipeline *completion*, not the scheduled start time. A 41-minute run starting at 03:00 UTC writes entries at 03:41 UTC — this is correct. Never assess legitimacy from `issued_at` alone; cross-check `pipeline_executions.started_at`.

**Prod-vs-Dev Compose Drift Trap:** Active stack is always `docker-compose.dev.yml`. If prod containers appear (`climate_db_prod`, `climate_frontend_prod`, `phase2-backend-N`), that is wrong state — stop them and restore dev compose. The startup script at `/opt/hewasense/start.sh` must always reference dev compose.

**Destructive Action Gate (ALL sensitive work):** Never execute a destructive or irreversible action (delete DB records, drop containers, wipe data, force-push, remove files) on user instruction alone. Required sequence: (1) verify the claim independently, (2) show the user what will be affected and the evidence, (3) if data is correct — push back explicitly and state why. Do not comply silently to avoid friction. This applies to shadow run DB, server infrastructure, git history, model artifacts, configs — anything hard or impossible to reverse.
