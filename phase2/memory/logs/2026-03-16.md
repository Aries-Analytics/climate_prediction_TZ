### /simplify Pass — Frontend Performance & Bug Fixes

Three review agents (reuse, quality, efficiency) ran in parallel against the dashboard optimization commit.

**Bugs found and fixed:**

1. **Cache inverted condition** — `response.statusText !== 'OK'` meant real axios responses were NEVER cached. Fixed via `_fromCache` sentinel on `InternalAxiosRequestConfig` (module augmentation). Request interceptor sets `config._fromCache = true` on cache hit; response interceptor skips re-caching when flag is set.

2. **Cache memory leak** — stale entries checked on read but never deleted → unbounded Map growth. Fixed with lazy-evict: `else { _responseCache.delete(key) }`.

3. **Plotly lazy-loaded** — `Chart.tsx` now uses `React.lazy(() => import('react-plotly.js'))` with Suspense fallback. 4.7 MB chunk no longer blocks initial render.

4. **GeographicMap lazy-loaded** — `TriggersDashboard.tsx` lazy-imports GeographicMap (1.97 MB Tanzania GeoJSON). Suspense fallback added.

5. **Column cascade recomputes** — `TriggersDashboard.columns` inline array recreated every render → cascaded filteredRows → sortedRows → paginatedRows recomputes. Wrapped in `useMemo([], [])`. `ModelPerformanceDashboard.metricsColumns` hoisted to module level.

6. **Missed axios imports** — `claimsService.ts` and `BacktestingValidation.tsx` still imported bare `axios`, bypassing auth interceptors and cache. Switched to `axiosInstance`.

### GOTCHA: Server DB Direct psql Trap

**Antipattern:** `psql -U postgres` or `psql -U climate_user` inside `climate_db_dev` → `FATAL: role does not exist`. Those roles don't exist — username is in `DATABASE_URL` env var.

**Correct method:**
```bash
docker exec climate_backend_dev python3 -c "
from app.core.database import SessionLocal
db = SessionLocal()
# your query here
db.close()
"
```
The ORM picks up `DATABASE_URL` from the container's environment automatically.
