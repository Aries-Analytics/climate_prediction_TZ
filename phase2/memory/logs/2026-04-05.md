**Key learnings (guardrails added)**:
- Harvard Dataverse `/api/access/datafile/{id}` returns 303 → S3 signed URL. Must follow redirect manually without carrying headers — S3 rejects auth headers in signature validation.
- Dryad `/api/v2/files/{id}/download` requires auth (401); `/downloads/file_stream/{id}` serves Cloudflare JS challenge. Not automatable without browser.
- MapSPAM 2020 yield unit is `mt/ha` (not `kg/ha` as older docs say) — do NOT divide by 1000.
- Windows cp1252 terminal: avoid Unicode arrows (→, ⟶) in print statements — use `->` instead.
