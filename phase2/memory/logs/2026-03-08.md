### Issues Diagnosed & Fixed

#### 5. 502 Bad Gateway — Frontend Down (vite.config.ts)
- **Symptom:** `GET /node_modules/.vite/deps/react_jsx-dev-runtime.js 502` — all dashboard routes returning gateway errors.
- **Root cause:** Previous session added `optimizeDeps.include: ['plotly.js', 'react-plotly.js']`. Vite's esbuild pre-bundler cannot resolve Node.js-specific `require('buffer/')` inside plotly → entire `.vite/deps` cache poisoned, breaking ALL modules including React itself.
- **Intermediate failure:** Added `optimizeDeps.exclude` → new error: `react-plotly.js does not provide an export named 'default'` — Vite served raw CJS without ESM conversion.
- **Fix:** Removed entire `optimizeDeps` block from `vite.config.ts`. Also cleared stale cache on server: `docker exec climate_frontend_dev rm -rf /app/node_modules/.vite`.
- **Rule:** `optimizeDeps.include` → esbuild (fails on Node.js CJS); `optimizeDeps.exclude` → raw CJS to browser (fails ESM). Vite default auto-detection is the only working option for react-plotly.js. `manualChunks` in `build.rollupOptions` is safe (Rollup-only, production builds).

#### 6. ExecutiveDashboard — 800ms Fake Loading Spinner
- **Root cause:** `useState(true) + setTimeout(800)` simulated async load for locally-computed `generateMockData()` — always synchronous, never an actual API call.
- **Fix:** `useMemo(() => generateMockData(selectedYear), [selectedYear])` — zero-delay. Real alerts fetch in independent `useEffect` on mount (parallel, non-cascading).
- **Rule:** Mock/local-computed data must never use async loading patterns. `useMemo` for derived state, `useEffect` for network calls.

#### 7. Historical Validation Tab — 500 Error (simulation.py:143)
- **Symptom:** Risk Management → Historical Validation tab returned 500: `"BacktestingService object has no attribute 'run_simulation'"`.
- **Fix:** `service.run_simulation(simulation_id)` → `service.run_phase_based_simulation(simulation_id)`. One-line fix.
- **Seeded:** Two simulations — ID 1 (out-of-sample 2000-2014, location_id=6) and ID 2 (in-sample 2015-2025, location_id=6). Dashboard fetches highest completed ID → ID 2.

#### 8. BacktestingValidation — Hardcoded Text Mismatches (commit `c82382d`)
- **Symptom 1:** Banner showed "backtested against 10 years" — simulation covers 2015–2025 = 11 years.
- **Symptom 2:** Chart title hardcoded "(2015-2025)" — won't adapt to different simulations.
- **Fix:** `"10 years"` → `{totalYears} years`; `"(2015-2025)"` → `({report.executive_summary.period})`.
