### 2. Sigmoid → Physical-Threshold CDF Probability Conversion (commit `fdb30b9`)
- **Symptom:** `sigmoid(z_score)` produced physically meaningless probabilities:
  - Wrong direction for drought (low rainfall → low P, should be high P)
  - All 4 trigger types got identical values from same raw input
  - No connection to calibrated Kilombero contract thresholds
- **Root cause:** Probability conversion was a mathematical convenience, not grounded in the two-layer architecture (Layer 1: ML prediction → Layer 2: Parametric threshold comparison)
- **Fix:** Replaced sigmoid with `_raw_to_probability()`:
  - Denormalise model z-score output → predicted mm (mean=79.15, std=77.06)
  - Determine Kilombero crop phase from target month (`get_kilombero_stage()`)
  - Compute `P = norm.cdf((phase_threshold - predicted_mm) / model_rmse_mm)` per trigger
  - `drought`: P(mm < phase.min) | `flood`: P(mm > phase.excessive) | `crop_failure`: P(mm < phase.min × 0.5) | `heat_stress`: drought proxy × 0.6
  - Phase-aware: flowering threshold (120mm) ≠ harvesting (80mm)
- **Files:** `backend/app/services/forecast_service.py`
- **Sources used:** `rice_thresholds.RAINFALL_THRESHOLDS` (TARI/FAO), `master_dataset.csv` stats, training RMSE=0.43z

## Learned Behaviors

- **Stale advisory lock = container restart required.** `pg_try_advisory_lock(123456)` is session-level on a dedicated raw connection — it survives ORM commits, transactions, and even `db.rollback()`. Only a container restart (which calls `_clear_stale_locks()` on startup) releases it. Signal that lock is stale: scheduler fires but immediately logs "lock already held" with no prior "Pipeline execution starting" log.
- **Sigmoid is wrong for directional probabilities.** `sigmoid(z_score)` gives 0.5 at z=0 (average rainfall) — this is neutral, not useful. For drought, low rainfall (negative z) gives LOW probability via sigmoid, but should give HIGH probability. Always use direction-aware CDF with a physical anchor point.
- **Two config trap: CLI vs VS Code extension.** `claude mcp list` (terminal) and `/mcp` (extension) reflect different session states. An MCP server connected in terminal is NOT automatically available in the extension. Project-scoped MCP entries in `.claude.json` must match the EXACT working directory path the extension opens.
