## Frontend Zone-Aware Dashboard Update

### Problem
Frontend EvidencePackDashboard showed only aggregate metrics with hardcoded zone text. No basis risk display, no GO/NO-GO gate rendering, no zone tabs. ForecastDashboard had hardcoded `kilomberoLocations` array with zone IDs, coordinates, and names baked into the code.

### Design Principle
Per GOTCHA: "we don't hardcode sections which should be data-driven." All zone names, coordinates, and counts come from the API — never from frontend constants.

### Changes Made

**`EvidencePackDashboard.tsx` (full rewrite):**
- Zone tabs ("All Zones" | per-zone) — zone list derived from `/execution-log` API response, not hardcoded
- KPI cards (Brier/RMSE/ECE/Total) update per selected tab — aggregate or zone-specific
- Basis risk section — fetches `/basis-risk`, shows per-zone or aggregate corroboration stats + gate status
- GO/NO-GO gates section — fetches `/final-report`, renders overall verdict + per-zone verdicts with pass/fail chips
- Confusion matrix cards switch per zone tab
- Forecast snapshot table shows Zone column (from API `region_id` → zone name lookup) when in aggregate view
- Shadow run progress info box is data-driven: zone count, names, and forecasts_per_day from the API response
- All 4 API endpoints now called: `/metrics`, `/execution-log`, `/basis-risk`, `/final-report`

**`ForecastDashboard.tsx`:**
- Removed hardcoded `kilomberoLocations` array
- Added `PilotLocation` interface + `pilotLocations` state
- Added `fetchPilotLocations()` — fetches from `/api/locations`
- `activePilotLocations` memo filters to only locations that appear in actual forecast data (truly data-driven)
- Added `locationId` to `Forecast` interface (was missing — backend sends it, frontend ignored it)
