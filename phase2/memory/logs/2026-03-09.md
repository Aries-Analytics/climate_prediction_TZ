## Issues Diagnosed & Fixed

### 1. ForecastLog.threshold_used Always NULL (commit `33dc78e`)
- **Symptom:** Every `forecast_logs` row had `threshold_used = NULL` — calibrated trigger thresholds never attached to shadow-run evidence records
- **Root cause:** `orchestrator.py:_generate_forecasts()` built `ForecastLog` objects without setting `threshold_used`. The calibrated `configs/trigger_thresholds.yaml` existed but was never referenced.
- **Fix:** Added `_PROB_THRESHOLDS` dict (drought/flood=0.65, heat_stress/crop_failure=0.60) directly in `_generate_forecasts()`. Each `ForecastLog` now receives `threshold_used=_PROB_THRESHOLDS.get(f.trigger_type)`.
- **File:** `backend/app/services/pipeline/orchestrator.py`

### 2. ForecastLog.forecast_distribution Always NULL (commit `33dc78e`)
- **Symptom:** JSON column always empty — no horizon tier, confidence bounds, or insurance eligibility in evidence pack
- **Root cause:** Same ForecastLog construction block never set `forecast_distribution`
- **Fix:** Populated with `{"horizon_tier", "is_insurance_trigger_eligible", "confidence_lower", "confidence_upper"}` using new `get_horizon_tier()` helper
- **File:** `backend/app/services/pipeline/orchestrator.py`

### 3. No Python API for Horizon Insurance Eligibility (commit `33dc78e`)
- **Symptom:** No programmatic way to distinguish 3-month (trigger-eligible) from 6-month (advisory) forecasts
- **Root cause:** Horizon tier concept existed in docs but had no code representation
- **Fix:** Added `HORIZON_TIERS = {3: "primary", 4: "primary", 5: "advisory", 6: "advisory"}` and `get_horizon_tier(horizon_months)` to `rice_thresholds.py`
- **File:** `backend/app/config/rice_thresholds.py`
