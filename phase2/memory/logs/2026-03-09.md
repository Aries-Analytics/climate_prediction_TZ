## Learned Behaviors (Session 2)

- **Postgres Fixture Trap:** Any local fixture calling `SessionLocal()` in a test = real Postgres, not conftest SQLite. Always use the conftest `db` fixture — never define a local one.
- **Hypothesis + function-scoped fixtures:** Must add `suppress_health_check=[HealthCheck.function_scoped_fixture]` to every `@settings` block when the test uses `db`, `caplog`, or any other function-scoped fixture.
- **SOURCE_MIN_WINDOW_DAYS mirror rule:** When testing `calculate_fetch_range`, test assertions must mirror the production `SOURCE_MIN_WINDOW_DAYS` override logic or they will fail for `era5`/`ocean_indices` sources.
- **MagicMock + SQLAlchemy = PendingRollback cascade:** Passing a MagicMock object to a DB column causes `ProgrammingError` which leaves the SQLAlchemy session in `PendingRollback` state. All subsequent examples in the same Hypothesis test then fail with `PendingRollbackError` (Flaky). Fix: ensure ALL DB-bound mock return values have concrete types.
- **UTC datetime in monitoring tests:** `MonitoringService` treats naive datetimes as UTC (`replace(tzinfo=timezone.utc)`). Tests creating `created_at=datetime.now()` on EAT (+3) machines will get `forecast_freshness_days = -1`. Always use `datetime.utcnow()` for test data that the monitoring service will read.
- **Query-based cleanup > object-reference cleanup in Hypothesis:** `db.delete(obj)` fails if the session is in PendingRollback state. Use `db.query(Model).filter(...).delete()` + `db.rollback()` at start of `finally` block.
