## Project Overview

**HewaSense** is a climate intelligence system for parametric crop insurance in Tanzania. It serves agricultural decisions in the Kilombero Basin, Morogoro region.

### Pilot Configuration
- **Region:** Morogoro (Kilombero Basin)
- **Location ID:** 6 (canonical — all components must use this)
- **Crop:** Rice
- **Farmers:** 1,000 pilot participants
- **Insurance type:** Parametric (triggers based on climate data, not loss assessment)

## Climate Data Sources

| Source | Variable | Resolution | Update Frequency |
|--------|----------|-----------|------------------|
| CHIRPS | Rainfall | 0.05° (~5km) | Pentadal (5-day) |
| NASA POWER | Temperature, Humidity, Solar | Point | Monthly |
| ERA5 | Temperature, Pressure, Wind, Soil Moisture | Grid | Monthly |
| MODIS (GEE) | NDVI, LST | 250m | ~16 days |
| Ocean Indices | ENSO (ONI), IOD (DMI) | Global | Monthly |

### Physical Bounds (Quality Gates)

- Precipitation: 0 – 500 mm/day
- Temperature: -10°C – 55°C
- NDVI: -1.0 – 1.0
- Relative Humidity: 0 – 100%

---

## Drought & Risk Indices

| Index | Definition | Trigger Threshold |
|-------|-----------|------------------|
| SPI (Standardized Precipitation Index) | Statistical measure of rainfall deficit | SPI < -1.0 = moderate drought |
| NDVI Anomaly | Deviation from long-term vegetation mean | < -0.15 = vegetation stress |
| Cumulative Rainfall Deficit | Below-average rain over rolling window | Configurable per contract |

---

## Parametric Insurance Logic

1. Climate data ingested → quality validated
2. Trigger conditions evaluated against thresholds
3. If trigger met → payout calculated automatically
4. Payout threshold: **75%** (canonical, in `state.json → shared_contract`)
5. No loss assessment needed — purely data-driven

## ML Model Architecture

- **Training pipeline:** `model_development_pipeline.py` → feature engineering via `preprocess.py`
- **Serving pipeline:** `forecast_service.py` → `prepare_features` → `.predict()`
- **Feature count:** 77 selected features (from ~594 engineered)
- **Model selection:** Driven by `outputs/models/active_model.json` (auto-generated by training pipeline)
- **Current best:** XGBoost (R²=0.840, 77 features)
- **Model method:** `.predict()` only (never `.predict_proba()`)
- **Feature schema:** `feature_schema.json` (canonical feature list)
- **No fallbacks:** If `active_model.json` or model file missing → fail explicitly (GOTCHA Law #1)
